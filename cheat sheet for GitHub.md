Работа в Git Bash
1. Для создания проекта необходимо выбрать директорию, в которой он будет располагаться на компьютере
   $ pwd - показать директорию в которой вы находитесь
   $ cd 'путь к директории' - выбрать нужную директорию
   $ cd ~ - перейти к домашней диретории
   
2. Создать рабочую директорию (папку), файлы на компьютере
   $ mkdir 'имя папки'- создать директорию (папку) 
   $ mkdir -p 'doc/doc2/doc3' - создать структуру директорий (папок)
   $ touch 'имя файла.расширение' - создать файл
   
3. Настроить Git для работы в команде
Чтобы участникам проекта было понятно кто вносил изменения, необходимо указать информацию о пользователе: имя и его электронную почту
   $ git config --global user.name 'имя пользователя'
   $ git config --global user.email 'электронная почта'
Все глобальные настройки Git хранит в файле .gitconfig в домашней директории. Команда запишет в этот файл указанные имя и почту. 
Вывести на экран содержимое файла: $ cat ~/.gitconfig (вы).
Другой способ проверки — вывести содержимое файла конфигурации: git config --list.

4. Сделать папку (проект) репозиторием для отслеживания изменений в проекте.
   - для этого необходимо перейти в нужную директорию (cd);
   - ввести команду git init;
При успешном выполнении: Initialized empty Git repository in C:/Users/katya/'имя пепозитория'/.git/
В подпапке .git Git будет хранить всю служебную информацию.
   $rm -rf .git - разгитить папку, т.е. удалить папку .git
Если удалить .git, то вся история проекта будет стёрта без возможности восстановления — останется только последняя версия файлов.
   $git status - проверить статус репозитория
   
5. Сделать файлы отслеживаемыми, добавить в репозиторий локально
   $git add --all  - сделать все файлы в папке отслеживаемыми в репозитории
   $git add 'имя файла.расширение' - сделать конкретный файл отслеживаемым в репозитории
   $git add . - добавить всю текущую папку в репозиторий (точка позволяет обратиться к текущей папке)
   
   !Проверка - $git status

Файлы, которые отмечены зелёным, теперь отслеживаются и готовы к сохранению. 
Но сохранения пока не произошло, потому что команда git add только запоминает текущее содержимое файла. 
После каждого внесения изменения необходимо вызывать команду git add.  

6. Сделать коммит файла(ов), т.е. сохранить изменение файла(ов) в истории.
   $git commit - выполнить коммит.
   $git commit -m 'сообщение'- выполнить коммит с сообщением
   
   !После коммита, команда git status перестанет выводить сообщение No commits yet (англ. «ещё нет коммитов»).
   
7. Просмотреть историю всех коммитов - $git log

8. Зарегистрироваться в Git Hub, если нет учётной записи

9. Создать удалённый репозиторий. Имя проекта и удалённого репозитория лучше делать одинаковыми.

10. Сгененировать ключи SSH
    * если есть, см. п 11
   $ls -la .ssh/  - проверить наличие ssh-ключей
Если есть файлы с расширением *.pub, то клбчи создавались.
   $ssh-keygen -t ed25519 -C 'электронная почта аккаунта на GitHub' - генерация ключей ssh.
или
   $ssh-keygen -t rsa -b 4096 -C 'электронная почта аккаунта на GitHub' - генерация ключей ssh.
! Появится сообщение: > Generating public/private rsa key pair. - сгенерированы публичный и приватный ключи.
Будет создано два файла — id_ed25519 и id_ed25519.pub (приватный и публичный).

Указать место хранения ключей 
> Enter a file in which to save the key 
например  (C:\Users\<имя_пользователя>\.ssh\):[Press enter] 

!Проверить наличие ключей ($ls -la .ssh/)

11. Привязать SSH-ключ к GitHub.

Скопировать содержимое файла с публичным ключом в буфер обмена:
   $clip < ~/.ssh/id_rsa.pub
или для ed25519:
   $clip < ~/.ssh/id_ed25519.pub
   
если clip не сработает:
   $cat ~/.ssh/id_rsa.pub 
или 
   $cat ~/.ssh/id_ed25519.pub
и скопируйте вывод в буфер обмена из консоли.

  Перейдите на GitHub и выберите пункт Settings (англ. «настройки») в меню аккаунта.
   - в меню слева нажмите на пункт SSH and GPG keys.
   - в открывшейся вкладке выберите New SSH key (англ. «новый SSH-ключ»).
   - в поле Title (англ. «заголовок») напишите название ключа. Например, Personal key (англ. «личный ключ»).
   - в поле Key type (англ. «тип ключа») должно быть Authentication Key (англ. «ключ аутентификации»).
   - в поле Key скопируйте ваш ключ из буфера обмена.
   - нажмите на кнопку Add SSH key (англ. «добавить SSH-ключ»).
   - проверьте правильность ключа с помощью следующей команды:  $ ssh -T git@github.com

12. Связать локальный и удалённый репозитории.
$git remote add - привязать удалённый репозиторий к локальному. 
Перейдите на страницу удалённого репозитория, выберите тип SSH и скопируйте URL. 
Откройте консоль, перейдите в каталог локального репозитория и введите команду git remote add (от англ. remote — «удалённый» и add — «добавить»).
$git remote add origin git@github.com:'имя аккаунта'/'имя удалённого репозитория'.git  
Команде необходимо передать два параметра: имя удалённого репозитория и его URL. 
В качестве имени используйте слово origin. А URL вы скопировали со страницы удалённого репозитория.

origin (англ. «источник») — стандартный псевдоним, с помощью которого можно обращаться к главному удалённому репозиторию 
(обычно такой репозиторий один). 

! Ctrl+Shift+V или ПКМ ? Paste - вставить текст из буфера обмена в командную строку.

13. Убедиться, что репозитории связаны
$ git remote -v. 
Должно быть в выводе две строки:
origin    git@github.com:%ИМЯ_АККАУНТА%/%ИМЯ-ПРОЕКТА%.git (fetch)
origin    git@github.com:%ИМЯ_АККАУНТА%/%ИМЯ-ПРОЕКТА%.git (push)

Прочие команды:
$pwd (от англ. print working directory, «показать рабочую папку») — узнать, в какой папке вы сейчас находитесь.
$cd ~ (от англ. change directory — «сменить директорию»)  - перейти к домашней директории (/Users/Username).
ls (от англ. list directory contents, «отобразить содержимое директории») — вывести содержимое директории, посмотреть список файлов/папок в директории;
ls -a  -  показать так же скрытые файлы и папки, названия которых начинаются с символа '.'.
ls ~ - вывести содержимое домашней директории вне зависимости от того, в какой диретории сейчас находитесь.
ls .. - показать содержимое родительской директории.
cd имя_папки (от англ. change directory — «сменить директорию») - сменить директорию, перейти в выбранную папку. Если в названии папки есть пробелы, при вводе нужно использовать кавычки: cd "имя_папки"
cd 'имя директории' (от англ. change directory, «сменить директорию») — перейти в необходимую директорию.
cd 'директория1'/'директория2' - перейти в диреторию2;
cd ..  - перейди на уровень выше, в родительскую папку;
cd / -  перейди в корневую директорию.
.. - вернуться в родительскую директорию — то есть на уровень выше, — вместо названия папки нужно написать две точки: ..
. - обратиться к текущей директории.

Работа с файлами
touch 'имя файла' - создать файл.
touch ../../'имя файла.расширение' - создаст файл на две папки выше по иерархии.
mkdir - создать директорию.
mkdir -p - создать структуру директорий.
mkdir ~/'имя папки' - создаст папку внутри домашней директории.

Копирование и перемещение
cp что_копируем куда_копируем (от англ. copy, «копировать») - копирование файлов.
cp что_копируем что_копируем что_копируем куда_копируем - копирование файлов.

mv 'имя файла,который хотим перемсетить'/.'путь, куда хотим переместить' (от англ. move, «переместить»)- перемещение файлов и папок.

Чтение
cat (от англ. concatenate and print — «объединить и распечатать») - вывести на экран содержимое файла.

Удаление
rm 'имя файла'- удалить файл.
rmdir 'имя папки' - удалить папку.
rm -r 'имя папки' (-r — от англ. recursive, «рекурсивный») - удалить папку со всем её содержимым.
&& - объединение сразу несколько команд.
?, ? - показать историю команд.
Tab - автозаполнение по первым символам.
/ + Tab (2 раза) - показать содержимое корневой директории.

Настройка Git. Прописать данные о себе
git config --global user.name 'имя пользователя' - инфорция для файла .gitconfig.
git config --global user.email 'электронная почта' - инфорция для файла .gitconfig.
git config --list  list (англ. «список») - вывести содержимое файла конфигурации Git.

Инициализируем репозиторий
git init (от англ. initialize — «инициализировать») - cделать папку репозиторием.
rm -rf .git - «разгитить» папку.
git status - проверить статус репозитория.

Добавить файлы в репозиторий
git add - подготовить файлы к сохранению.
git add --all - подготовить все файлы директории к сохранению.
git add 'название файла' - подготовить файлы к сохранению конкретный файл.

Выполнить коммит в репозиторий
git commit  - сохранение изменения в репозиторий.
git commit -m ‘сообщение’ - сохранение изменения в репозиторий с сообщением.
git log (от англ. log — «журнал [записей]») - просмотреть историю коммитов.

Генерация SSH-ключа
ssh-keygen -t ed25519 -C 'электронная почта на GitHub'. 
или
ssh-keygen -t rsa -b 4096 -C 'электронная почта на GitHub'.
ls -a ~/.ssh - проверка генерации ключа.
ssh -T git@github.com - проверьте правильность ключа.

Связать локальный и удалённый репозитории.
git remote add - привязать удалённый репозиторий к локальному.
